Hermes系统规划（落地版）

# 文档概述

## 1.1 文档目的

本文档明确智能化营销推送系统的建设目标、功能、规则及验收标准，作为研发落地统一依据，助力游戏营销由人工粗放运营升级为智能精准运营，适配多游戏 App 接入与全生命周期营销场景。

## 1.2 建设范围  

系统结合全链路营销逻辑，覆盖游戏营销全流程，支持多 App 批量接入、零代码配置推送，覆盖用户全生命周期营销场景，依托 FCM 通道跨平台推送。

#1项目背景与建设目标

## 2.1 项目背景

数据时效性差以及无行为触发的自动化个性化推送，错失关键营销时机，影响用户留存与付费转化；

缺少游戏用户精准画像与分层体系，推送同质化严重，未结合玩家行为、等级、付费偏好，导致转化低、用户骚扰率高；

营销推送无数据闭环，难以复盘优化与沉淀可复用策略，需搭建数据驱动运营闭环；

## 2.2 核心目标

精准化：构建贴合游戏场景的用户画像体系，支持多维度智能圈群，结合玩家等级、行为、付费偏好实现精准推送。

自动化：搭建全场景自动化推送引擎，支持定时、事件、循环触发及A/B测试，降低人工成本，适配高频场景化推送。

数据化：实现推送全链路数据实时监控，自动生成多维度复盘报告，构建数据驱动的运营飞轮。

高效化：支持零代码配置活动，多游戏批量接入与独立管控，大幅降低运营人力成本。

智能化：具备智能推荐、高潜用户优选、效果预测、异常告警、智能频控等能力，提升决策与推送效率。

# 核心用户与使用场景

## 3.1 核心用户

游戏运营/增长：负责游戏推送活动配置、内容编辑、效果查看与日常运营，满足新手引导、活动推广、召回等场景化推送需求。

游戏运营负责人：负责推送审批、营销效果监控，制定专属策略，统筹多游戏运营与全局数据管控。

数据分析师：负责推送数据导出、转化分析，输出人群与内容优化建议，提升推送 ROI。

系统管理员：负责账号权限、渠道配置、游戏接入审核及系统运维，保障系统稳定与数据安全。

## 3.2 典型业务场景

新玩家注册后，自动推送新手引导与礼包，助力新手激活。

对未购道具、未完成充值/通关的玩家，自动推送激励召回内容，提升转化。

按生日、纪念日、节日自动推送专属营销内容与权益，增强用户粘性。

对沉默7/15/30天玩家分层召回，推送个性化回归礼包与内容，唤醒流失用户。

版本更新、新副本、限时活动上线时，同步触达。

开展推送内容与人群A/B测试，自动统计数据并全量投放最优策略。

对高潜付费玩家推送专属活动与道具，对低活跃付费用户推送福利，提升付费转化。

6.1 智能化营销（任务配置）

6.1.1 运营活动

将零散无序的推送任务打包成有组织标准化的营销活动以及模板，如：即时活动、定时活动、触发活动、A/B测试活动、游戏专属活动

6.1.2 触发规则

时间触发：指定时间单次执行，或按日、周、月等周期循环执行（签到、福利、活动等场景）

行为触发：依注册、登录、浏览、购买、充值、通关、社交等行为触发推送

规则触发：根据标签、分群变、付费/等级/战力达标、流失风险变化等规则触发推送

6.1.3 执行策略

延迟执行：触发后可延迟指定时长推送，精准把握营销时机

循环执行：支持按日 / 周 / 月周期循环，满足常态化营销需求

终止条件：玩家完成转化行为后自动停止触达，支持多条件配置

A/B 测试：支持分流、赛马模式，自动统计效果并择优全量投放

频控限制：可限制玩家参与次数，避免过度推送

排除互斥：设置互斥任务，防止重复触达

流程画布：拖拽配置多节点、多渠道复杂推送流程，适配游戏复杂运营场景，如：用户的全生命周期运营

6.1.4 推送审批

支持审批流转可视化，运营人员可实时查看审批进度，及时跟进审批流程，确保活动按时上线以及推送安全。

6.1.5 其他

全生命周期管理：覆盖活动创建、预览、审批、执行、监控、启停至结束，形成完整运营闭环

快捷操作：支持复制、编辑、启停等，按游戏 App 隔离管理

日志管理：全量记录操作信息，可追溯、便于审计排查

优先级设置：紧急活动可设高优先级，保障推送及时触达

6.2 素材管理

支持富文本、图片、URL、视频、音频以及变量填充等多种素材管理

打通cms素材管理，提升业务内容配置效率

搭建游戏专属模板库，涵盖新手、召回、更新等模板，可复用编辑，提升制作效率

提供 AI 智能内容生成，支持批量差异化创作、仿写衍生，降低运营成本

内置游戏专属敏感词库，自动过滤违规内容，保障推送合规

6.3 用户圈选

6.3.1 标签配置

标签创建：支持灵活的标签创建和管理

ID 标签：通过导入ID方式创建

规则标签：通过通过埋点事件/属性规则化创建

首末次标签：指定时间内用户完成首次/末次事件的属性作为标签

指标值标签：指定时间内用户完成首次/末次事件的指标聚合值作为标签

SQL标签：通过SQL创建的标签

标签类型管理

支持用户对标签进行分类：基础标签、行为标签、交易标签、游戏专属标签等

6.3.2 分群配置

静态圈群：可通过埋点事件/属性规则化创建、批量上传玩家ID列表等方式，创建固定玩家群体（如特定服务器玩家、高付费玩家）。

动态分群：可通过埋点事件/属性规则化创建，并自动更新（如实时更新沉默7天玩家、新注册玩家）。

模型分群：通过Lookalike相似人群拓展算法，基于种子人群包（如高付费玩家、高活跃玩家）进行同质扩样，自动识别相似玩家群体，提升推送精准度。

SQL分群：通过SQL创建的分群

6.3.3 玩家画像

整合玩家标签、行为轨迹、推送及付费记录，实现信息可视化；支持多维度快速检索，便于精准制定推送策略；可分析全链路行为，识别运营瓶颈，优化推送时机与内容。

6.4 数据洞察

6.4.1 核心埋点覆盖

触达指标：涵盖覆盖人数、发送数、成功数、成功率、到达率、授权率等核心触达数据。

效果指标：涵盖点击 / 点击率、打开率、跳转率、游戏内跳转率、内容互动率等核心效果数据。

转化指标：回归数 / 率、充值数 / 金额、付费转化率、道具购买数、关卡通关率、活动参与率等游戏专属转化数据

玩家行为指标：在线时长、登录频次、多日留存（1/7/30 日）、LTV，支撑游戏运营优化

6.4.2 数据可视化

营销大盘：集中展示项目内整体推送任务数据可视化，便于全局管控。

单活动详情：提供单任务实时数据趋势、分渠道 / 人群对比及转化漏斗分析，支持定位推送失败原因。

自动报告：支持配置化指标创建，自动生成日报、周报及活动复盘报告，包含核心数据与优化建议，支持自定义模板，降低人工成本。

数据钻取：支持从宏观数据下钻至玩家行为全链路分析，结合画像定位问题，优化推送策略。

6.5 系统管理模块

6.5.1 App 接入

提供配置化接入能力，支持游戏 App 自动化接入与独立管控，完成系统与游戏客户端的对接（如 FCM 通道绑定、行为数据上报协议配置），为后续用户触达、数据采集建立基础连接。

6.5.2 权限管理

依赖 IAM

6.5.3 审批管理

负责推送任务全流程审批：支持按活动类型、覆盖人数、推送频次等维度配置差异化审批流程，记录审批意见与进度，实现审批可追溯，保障推送活动合规性，避免违规推送与用户过度骚扰。

6.5.4 告警中心

提供推送任务监控告警配置：支持对推送执行状态、送达率、异常触发等设置告警规则，实时感知系统故障、任务失败、用户投诉等风险，及时推送告警通知，保障推送服务稳定性与业务连续性。
