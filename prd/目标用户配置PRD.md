# 目标用户配置 PRD

## 1. 功能概述

目标用户配置是推送任务创建流程中的第二步（Step 2），用于定义推送任务的受众范围。运营人员通过添加用户属性筛选条件，精确圈选目标推送人群。若未添加任何筛选条件，系统默认推送全量用户。

---

## 3. 运算符

根据字段数据类型，系统自动匹配可用运算符：

### 3.1 文本 / 枚举类型

| 运算符 | 标识 |
|-------|------|
| 等于 | `=` |
| 不等于 | `!=` |
| 包含 | `contains` |
| 不包含 | `not_contains` |

### 3.2 数值类型

| 运算符 | 标识 |
|-------|------|
| 等于 | `=` |
| 不等于 | `!=` |
| 大于 | `>` |
| 小于 | `<` |
| 大于等于 | `>=` |
| 小于等于 | `<=` |
| 包含 | `contains` |
| 不包含 | `not_contains` |

### 3.3 分群类型

| 运算符 | 标识 |
|-------|------|
| 属于分群 | `in_segment` |
| 不属于分群 | `not_in_segment` |

## 4. 筛选条件结构

### 4.1 数据模型

每条筛选条件的数据结构如下：

```json
{
  "id": "string",
  "field": "string",
  "operator": "string",
  "value": "string",
  "logic": "and | or",
  "children": [
    {
      "id": "string",
      "field": "string",
      "operator": "string",
      "value": "string",
      "logic": "and | or"
    }
  ]
}
```

### 4.2 层级关系

系统支持两级筛选条件嵌套：

- **一级条件（父级）**：顶层筛选行，支持添加子条件
- **二级条件（子级）**：挂载在某个父级条件下，不可再嵌套

### 4.3 逻辑关系

- **一级条件之间**：统一使用同一逻辑关系（且/或），点击逻辑标签可全局切换
- **二级条件之间**：同一父级下的子条件统一使用同一逻辑关系（且/或），点击逻辑标签可切换该组内的逻辑
- 默认逻辑关系为 `且（and）`

## 5. 交互设计

### 5.1 页面布局

```
┌─────────────────────────────────────────────────┐
│  用户属性          + 添加属性                      │
├─────────────────────────────────────────────────┤
│                                                 │
│  （无筛选条件时显示提示文案）                        │
│  "未添加筛选条件，将默认推送全量用户"                 │
│                                                 │
│  ┌── 且/或 ──┐                                   │
│  │  [字段▾] [运算符▾] [值输入] [+添加] [删除]      │
│  │    ┌── 且/或 ──┐                              │
│  │    │  [字段▾] [运算符▾] [值输入] [删除]         │
│  │    │  [字段▾] [运算符▾] [值输入] [删除]         │
│  │    └──────────┘                               │
│  │  [字段▾] [运算符▾] [值输入] [+添加] [删除]      │
│  └───────────┘                                   │
│                                                 │
│  [预估人数]    预估用户数：12,345                   │
└─────────────────────────────────────────────────┘
```

### 5.2 操作说明

| 操作 | 说明 |
|-----|------|
| 添加属性 | 点击「+ 添加属性」在一级条件列表末尾新增一行，默认运算符为 `=`，逻辑为 `and` |
| 添加子条件 | 点击一级条件行的「+添加」按钮，在该条件下新增一条二级子条件 |
| 删除条件 | 点击「删除」移除当前行（一级或二级） |
| 切换逻辑 | 点击竖线上的「且/或」标签，切换对应层级所有条件的逻辑关系 |
| 切换字段 | 切换字段时自动重置运算符（分群类型重置为 `in_segment`，其他重置为 `=`） |

### 5.3 字段选择器

字段选择器为自定义下拉面板，支持以下功能：

- **搜索**：支持按字段名称或标识模糊搜索
- **分类 Tab**：全部 / 用户属性 / 用户分群 / 用户标签
- **收藏**：点击星标收藏常用字段，收藏列表置顶显示，持久化到 `localStorage`
- **类型标识**：每个字段右侧显示数据类型图标和文字标签

### 5.4 值输入

- 枚举类型字段（如「目标平台」）：渲染为下拉选择框，选项由字段元数据定义
- 其他类型字段：渲染为文本输入框

## 6. 冲突检测

系统自动检测筛选条件中的逻辑冲突，规则如下：

- 当同一字段存在多个 `=`（等于）条件，且这些条件之间为 `且（and）` 关系，且值不相同时，判定为冲突
- 冲突时在筛选区域上方显示警告提示：「筛选条件逻辑冲突，请修改」

---

## 7. 预估人数

### 7.1 功能说明

点击「预估人数」按钮，系统根据当前筛选条件调用后端接口，返回符合条件的预估用户数量。

### 7.2 接口定义

```
POST /api/v1/push-tasks/estimate
```

请求参数：

```json
{
  "project_id": 123,
  "filters": [
    {
      "field": "country",
      "operator": "=",
      "value": "CN",
      "logic": "and",
      "children": [
        {
          "field": "platform",
          "operator": "=",
          "value": "iOS",
          "logic": "and"
        }
      ]
    }
  ]
}
```

响应：

```json
{
  "estimated_count": 12345
}
```

### 7.3 交互细节

- 按钮点击后显示 loading 状态，文案变为「计算中...」
- 未选择项目时提示「未选择项目，请先选择项目空间」
- 请求异常时提示「网络异常，请重试」
- 仅提交已填写完整（field 和 value 非空）的筛选条件
- 预估结果以千分位格式展示在按钮右侧

## 8. 提交数据结构

目标用户配置在任务提交时序列化为以下格式，存储在 `target_user_config` JSONB 字段中：

```json
{
  "filters": [
    {
      "field": "country",
      "operator": "=",
      "value": "CN",
      "logic": "and"
    }
  ]
}
```

## 9. 校验规则

- 目标用户步骤无必填校验，未添加筛选条件时默认推送全量用户
- 冲突检测为前端实时提示，不阻断提交
